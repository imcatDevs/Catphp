Csrf

Cat\Csrf — CSRF 방어
csrf()

Cross-Site Request Forgery 공격을 방어합니다. 세션 기반 토큰을 생성하고, 폼 제출 시 토큰 일치 여부를 검증합니다.

field()로 hidden input을 자동 생성하고, middleware()로 POST/PUT/PATCH/DELETE 요청에 자동 검증을 적용할 수 있습니다. 토큰은 세션에 저장되며, per-request XOR 마스킹으로 BREACH 공격을 방어합니다.

전체 메서드 레퍼런스
메서드반환설명
token()string원본 CSRF 토큰 반환 (없으면 생성)
maskedToken()string마스킹된 토큰 (BREACH 방어, 매 호출 다른 값)
field()string<input type="hidden"> HTML (마스킹된 토큰 사용)
verify()bool토큰 검증 (마스킹 + 원본 모두 지원)
middleware()callable자동 검증 미들웨어 (실패 시 403)
폼에서 사용
// 뷰 템플릿 — hidden 필드 자동 삽입
<form method="POST" action="/contact">
    <?= csrf()->field() ?>
    <input name="message" />
    <button type="submit">전송</button>
</form>

// 핸들러에서 수동 검증
if (!csrf()->verify()) {
    json()->fail('CSRF 토큰 불일치', 403);
}
미들웨어로 자동 적용
// 글로벌 미들웨어 등록 — 모든 변경 요청에 자동 검증
router()->use(csrf()->middleware());

// AJAX에서는 헤더로 전송
// fetch('/api/data', { headers: { 'X-CSRF-Token': token } })
주의: CSRF 미들웨어는 GET/HEAD/OPTIONS 요청은 건너뛰고, POST/PUT/PATCH/DELETE만 검증합니다. API 전용 라우트에서는 JWT 인증으로 대체하는 것이 일반적입니다.
BREACH 방어: field()와 maskedToken()은 매 호출마다 랜덤 XOR 마스크를 생성하여 토큰 값을 변경합니다. HTTPS + gzip 환경에서 응답 압축률 차이로 토큰을 추출하는 BREACH 공격을 방지합니다. verify()는 마스킹된 토큰과 원본 토큰 모두 검증합니다.
관련: Guard Auth
