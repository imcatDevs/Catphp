Cat\Request — HTTP 요청 추상화
request()

HTTP 요청 추상화 도구입니다. GET/POST/JSON 입력, 파일 업로드, 헤더, 메서드, 클라이언트 정보를 통합 API로 제공합니다.

input()은 GET → POST → JSON body 순으로 값을 찾습니다. _method 필드로 PUT/PATCH/DELETE 메서드 오버라이드를 지원합니다.

주요 메서드
메서드반환설명
input(string $key, mixed $default)mixedGET+POST+JSON 통합 입력
query / post / jsonmixed소스별 입력
all()array모든 입력
only(array $keys) / except(array $keys)array필터링
has / filledbool존재 / 비어있지 않음
file(string $key) / hasFile(string $key)—파일 업로드
method / isGet / isPost—HTTP 메서드
header / headers / bearerToken—요청 헤더
path / url / fullUrlstring요청 URL
isJson / isAjax / isSecurebool요청 타입 확인
ip / userAgent / referer—클라이언트 정보
사용 예제
// 입력 데이터
$name  = request()->input('name');
$page  = request()->query('page', '1');
$data  = request()->only(['name', 'email']);

// API 요청 판별
if (request()->wantsJson()) {
    json()->ok($result);
}

// Bearer 토큰
$token = request()->bearerToken();
IP 감지: ip()는 Ip 도구가 로드되어 있으면 trusted_proxies 설정을 존중하여 위임합니다. Ip 도구가 없으면 REMOTE_ADDR만 사용합니다 (프록시 헤더 스푸핑 방지).
보안: isSecure()는 X-Forwarded-Proto 헤더를 config('request.trusted_proxies')에 설정된 IP에서만 신뢰합니다. bearerToken()은 Apache CGI/FastCGI 환경에서도 Authorization 헤더를 감지합니다.
