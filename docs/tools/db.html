Cat\DB — PDO 쿼리 빌더
db()

PDO 기반의 경량 쿼리 빌더입니다. 지연 연결(Lazy Connection) 방식으로 실제 쿼리가 실행될 때만 DB에 연결하므로, DB를 사용하지 않는 요청에서는 연결 오버헤드가 전혀 없습니다.

모든 쿼리는 prepared statement를 사용하여 SQL Injection을 원천 차단하며, 테이블·컬럼명에도 validateIdentifier() 정규식 검증을 수행합니다. 체이닝 API로 직관적인 쿼리 작성이 가능하고, 복잡한 쿼리는 raw()로 직접 실행할 수 있습니다.

설정 — config/app.php
'db' => [
    'driver'   => 'mysql',          // mysql | sqlite | pgsql
    'host'     => '127.0.0.1',
    'port'     => 3306,
    'database' => 'catphp',
    'username' => 'root',
    'password' => '',
    'charset'  => 'utf8mb4',
    'options'  => [],               // PDO 추가 옵션
]

whereIn(string $col, array $vals)selfIN 절 조건 (배열 바인딩)
whereNull(string $col)selfIS NULL 조건
whereNotNull(string $col)selfIS NOT NULL 조건
orderBy(string $col, string $dir)self정렬. 'ASC' 또는 'DESC'
limit(int $n)self결과 개수 제한
offset(int $n)self결과 시작 위치

기본 CRUD
// SELECT — 단건 조회
$user = db()->table('users')->where('id', 1)->first();

// SELECT — 조건 + 정렬 + 페이징
$posts = db()->table('posts')
    ->select('id', 'title', 'created_at')
    ->where('status', 'published')
    ->orderBy('created_at', 'DESC')
    ->limit(10)->offset(0)
    ->all();

// INSERT
$id = db()->table('users')->insert([
    'name'  => 'Cat',
    'email' => 'cat@php.dev',
]);

// UPDATE
db()->table('users')->where('id', 1)->update(['name' => 'Updated']);

// DELETE
db()->table('users')->where('id', 99)->delete();
고급 쿼리
// OR 조건 + IN 절
$rows = db()->table('users')
    ->where('role', 'admin')
    ->orWhere('role', 'moderator')
    ->all();

$rows = db()->table('users')->whereIn('id', [1, 2, 3])->all();

// 트랜잭션 — 예외 시 자동 롤백
db()->transaction(function() {
    db()->table('accounts')->where('id', 1)->update(['balance' => 100]);
    db()->table('accounts')->where('id', 2)->update(['balance' => 200]);
});

// Raw SQL — 복잡한 JOIN 등
$stmt = db()->raw(
    'SELECT u.*, COUNT(p.id) as post_count FROM users u LEFT JOIN posts p ON u.id = p.user_id GROUP BY u.id',
    []
);
보안 기능: 연산자 화이트리스트(=, <, >, <=, >=, !=, LIKE, IN)와 validateIdentifier()로 테이블·컬럼명에 대한 SQL Injection을 차단합니다. where 없이 update()/delete()는 실행되지 않습니다.
팁: db()는 싱글턴이므로 체이닝 후 반드시 first()/all()/insert() 등으로 실행해야 다음 쿼리에서 이전 조건이 초기화됩니다.
