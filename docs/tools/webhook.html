Webhook

Cat\Webhook — Webhook 발송/수신 + HMAC 서명
webhook()

Webhook 발송 및 수신 도구입니다. HMAC-SHA256 서명으로 페이로드 무결성을 보장하며, 재시도 로직과 로깅을 내장합니다.

발송 시 URL은 http/https만 허용됩니다 (SSRF 방어). 서명 비교는 hash_equals()로 타이밍 공격을 방지합니다. GitHub 스타일 sha256=... 접두사도 자동 처리합니다.

발송 메서드
메서드반환설명
to(string $url)self대상 URL (http/https만)
secret(string $secret)selfHMAC 시크릿 (서명 자동 첨부)
payload(array $data)selfJSON 페이로드
header(string $name, string $value)self커스텀 헤더 추가
send()WebhookResult발송 실행 (재시도 포함)
수신 메서드
메서드반환설명
receive()selfphp://input에서 body + 서명 헤더 읽기
isValid(?string $secret = null)bool서명 검증 (hash_equals)
getPayload()?array수신 페이로드 (JSON 디코딩, 1MB/깊이32 제한)
getRawBody()string수신 원본 body
WebhookResult 메서드
메서드반환설명
status()intHTTP 상태 코드
body()string응답 body
ok()bool성공 여부 (2xx)
attempts()int시도 횟수
json()?array응답 JSON 디코딩
발송 예제
// 기본 발송
$result = webhook()->to('https://example.com/hook')
    ->payload(['event' => 'user.created', 'data' => $user])
    ->send();

if ($result->ok()) {
    // 성공 (2xx)
}

// HMAC 서명 + 커스텀 헤더
$result = webhook()->to('https://api.example.com/webhook')
    ->secret('my-webhook-secret')
    ->header('X-Event-Type', 'order.paid')
    ->payload(['order_id' => 123, 'amount' => 9900])
    ->send();
수신 검증 예제
// Webhook 수신 엔드포인트
router()->post('/api/webhook', function() {
    $wh = webhook()->receive();

    if (!$wh->isValid('my-secret')) {
        json()->fail('서명 검증 실패', [], 401);
    }

    $data = $wh->getPayload();
    // $data['event'], $data['data'] 처리...
    json()->ok(['received' => true]);
});
설정
// config/app.php
'webhook' => [
    'secret'      => '',      // 기본 HMAC 시크릿
    'timeout'     => 10,      // HTTP 타임아웃 (초)
    'retry'       => 0,       // 실패 시 재시도 횟수
    'retry_delay' => 1,       // 재시도 간 대기 (초)
    'log'         => false,   // 발송/수신 로깅
],
보안: 프로덕션에서는 반드시 시크릿을 설정하고, 수신 엔드포인트에서 isValid()로 서명을 검증하세요. 시크릿은 환경변수(env('WEBHOOK_SECRET'))로 관리하는 것을 권장합니다.
관련: Http Json Encrypt
