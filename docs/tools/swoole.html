Swoole

Cat\Swoole — 고성능 비동기 HTTP/WebSocket 서버
swoole()

Swoole 확장 기반 상주 프로세스 서버입니다. CatPHP Router와 자동 통합되며, 코루틴·연결 풀·태스크 워커·WebSocket·Hot Reload를 지원합니다.

PHP-FPM 대비 10~100배 높은 처리량을 달성합니다. 요청마다 프레임워크를 재부팅하지 않아 메모리와 I/O를 절약합니다.

4대 원칙: ⚡ 빠른 속도 — 상주 프로세스, require/config 1회 부팅, 연결 풀 재사용 | 🔧 사용 편리 — swoole()->http()->start() 한 줄로 시작 | 📖 쉬운 학습 — http/websocket/task/co 4가지 핵심 개념 | 🔒 보안 — 요청 격리(슈퍼글로벌 초기화), Graceful Shutdown, input() 살균 자동 적용

HTTP 서버 — 기본 사용
// swoole-server.php (또는 cli.php에서 swoole:start)
swoole()
    ->http()
    ->listen('0.0.0.0', 9501)
    ->workers(4)
    ->onBoot(function() {
        // CatPHP 라우트 자동 통합
        router()->get('/', fn() => 'Hello from Swoole!');
        router()->get('/api/users', fn() => json()->ok(db()->table('users')->get()));
    })
    ->start();
WebSocket 채팅 — 룸 기반
swoole()
    ->websocket()
    ->onWsOpen(fn($fd) => swoole()->join($fd, 'lobby'))
    ->onWsMessage(function(int $fd, string $data) {
        $msg = json_decode($data, true);

        match($msg['type'] ?? '') {
            'join'  => swoole()->join($fd, $msg['room']),
            'chat'  => swoole()->toRoom($msg['room'], [
                'from' => $fd, 'text' => $msg['text']
            ]),
            default => swoole()->push($fd, ['error' => 'unknown type']),
        };
    })
    ->onWsClose(fn($fd) => swoole()->leaveAll($fd))
    ->start();
태스크 워커 — 비동기 이메일 발송
// 태스크 핸들러 등록 (onBoot 내부)
swoole()->handle('email', function(array $payload) {
    mailer()->to($payload['to'])
        ->subject($payload['subject'])
        ->body($payload['body'])
        ->send();
    return true;
});

// 요청 핸들러에서 비동기 태스크 전송
router()->post('/api/register', function() {
    // DB 저장...
    swoole()->task('email', [
        'to'      => input('email'),
        'subject' => '가입 환영!',
        'body'    => '환영합니다.',
    ]);
    json()->ok(['registered' => true]);
});
코루틴 병렬 실행 + 연결 풀
// onBoot에서 연결 풀 생성
swoole()->createDbPool(8);

// 코루틴 내에서 연결 풀 사용
swoole()->co(function() {
    $pdo = swoole()->poolGet('db');
    try {
        $stmt = $pdo->prepare('SELECT * FROM users WHERE id = ?');
        $stmt->execute([1]);
        $user = $stmt->fetch();
    } finally {
        swoole()->poolPut('db', $pdo); // 반드시 반환!
    }
});

// 병렬 실행 (여러 API 동시 호출)
$results = swoole()->parallel([
    'users'  => fn() => http()->get('https://api.example.com/users'),
    'orders' => fn() => http()->get('https://api.example.com/orders'),
]);
CLI 명령어
# HTTP 서버 시작
php cli.php swoole:start
php cli.php swoole:start --port=8080 --host=127.0.0.1

# WebSocket 서버
php cli.php swoole:start --type=websocket

# 데몬 모드
php cli.php swoole:start --daemon

# 서버 관리
php cli.php swoole:status
php cli.php swoole:reload   # 워커 리로드 (코드 반영)
php cli.php swoole:stop     # Graceful Shutdown
설정
// config/app.php
'swoole' => [
    'host'             => '0.0.0.0',
    'port'             => 9501,
    'worker_num'       => 0,          // 0 = CPU 코어 수 자동
    'task_worker_num'  => 4,
    'max_request'      => 10000,      // 워커당 최대 요청 (메모리 누수 방지)
    'enable_coroutine' => true,
    'hot_reload'       => false,      // 개발 전용
    'ssl_cert'         => '',
    'ssl_key'          => '',
    'pool'             => ['db' => 0, 'redis' => 0],
],
요구 사항: Swoole 확장이 필요합니다 (pecl install swoole). OpenSwoole도 호환됩니다. PHP-FPM 환경에서는 사용하지 마세요.
보안 — 요청 격리: 매 요청마다 $_GET, $_POST, $_SERVER, $_COOKIE를 초기화하고 input() 캐시를 리셋합니다. 상주 프로세스에서 이전 요청 데이터가 남지 않습니다. 연결 풀 사용 시 반드시 poolPut()으로 반환하세요.
