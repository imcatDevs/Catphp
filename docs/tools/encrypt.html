Encrypt

Cat\Encrypt — Sodium 암호화
encrypt()

Sodium(libsodium) 기반 대칭 키 암호화와 HMAC-SHA256 서명을 제공합니다. seal()/open()은 데이터를 암호화/복호화하고, sign()/verify()는 데이터 무결성을 검증합니다.

암호화 키가 비어 있으면 RuntimeException을 던져 안전하지 않은 암호화를 방지합니다. 키는 __destruct()에서 sodium_memzero()로 메모리에서 안전하게 삭제됩니다.

설정 — config/app.php
'encrypt' => [
    'key' => 'your-32-byte-secret-key',  // 필수 (32바이트 권장)
]
전체 메서드 레퍼런스
메서드반환설명
seal(string $data)stringSodium 암호화 (nonce 자동 생성, base64)
open(string $encrypted)?string복호화. 실패 시 null
sign(string $data)stringHMAC-SHA256 서명 생성
verify(string $data, string $sig)bool서명 검증 (hash_equals 사용)
암호화 / 복호화
// 민감 데이터 암호화
$encrypted = encrypt()->seal('주민등록번호: 900101-1234567');

// 복호화
$plain = encrypt()->open($encrypted);
if ($plain === null) {
    // 복호화 실패 (키 변경 또는 데이터 변조)
}
서명 / 검증
// 데이터 무결성 서명 (웹훅 등)
$data = json_encode(['order_id' => 123]);
$sig  = encrypt()->sign($data);

// 수신 측 검증
if (encrypt()->verify($data, $sig)) {
    // 무결성 확인됨
}
보안: Sodium은 nonce를 자동 생성하므로 같은 데이터를 암호화해도 매번 다른 결과가 나옵니다. #[\SensitiveParameter]로 키가 스택 트레이스에 노출되지 않습니다.
관련: Auth Cookie
