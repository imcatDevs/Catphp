Cat\Json — JSON 응답 (CatUI 통합 포맷)
json()

API 응답을 CatUI APIUtil 호환 형식으로 출력합니다. ok(), fail(), error()로 일관된 응답 구조를 유지하며, paginated()로 페이지네이션 응답을 생성합니다.

응답 형식: {"success": bool, "statusCode": int, "data": ..., "message": "...", "error": ..., "timestamp": int}

전체 메서드 레퍼런스
메서드반환설명
ok(mixed $data, string $msg, int $code)never성공 응답 (200)
created(mixed $data, string $msg)never생성 성공 (201)
noContent()neverNo Content (204)
fail(string $msg, int $code, ?array $details)never검증 실패 (422)
error(string $msg, int $code, string $type)never시스템 에러 (500)
unauthorized(string $msg)never인증 필요 (401)
forbidden(string $msg)never권한 없음 (403)
notFound(string $msg)never찾을 수 없음 (404)
paginated(array $items, int $page, int $limit, int $total)never페이지네이션 응답
send(int $code, array $payload)never커스텀 JSON 응답
사용 예제
// 성공 응답
json()->ok(['user' => ['id' => 1, 'name' => 'Cat']]);
// → {"success":true, "statusCode":200, "data":{...}, "message":"Success", ...}

// 201 Created
json()->created(['id' => 123]);

// 검증 실패 + 상세 정보
json()->fail('유효성 검증 실패', 422, $v->errors());

// 편의 메서드
json()->unauthorized();    // 401
json()->forbidden();       // 403
json()->notFound();        // 404

// 페이지네이션 응답
$posts = db()->table('posts')->limit(10)->offset(0)->all();
$total = db()->table('posts')->count();
json()->paginated($posts, 1, 10, $total);
// → data: {items: [...], pagination: {page, limit, total, totalPages, hasNext, hasPrev}}
CatUI 통합: CatPHP json()과 CatUI APIUtil이 동일한 응답 형식을 사용하여 프론트엔드-백엔드 통합이 용이합니다.
주의: 모든 응답 메서드는 내부에서 exit을 호출합니다. 이후 코드는 실행되지 않으므로 핸들러의 마지막에 호출하세요.
