Cat\Json — JSON 응답 빌더
json()

API 응답을 통일된 JSON 형식으로 출력합니다. ok(), fail(), error()로 일관된 응답 구조를 유지하며, paginated()로 페이지네이션 응답을 생성합니다.

모든 응답 메서드는 내부에서 Content-Type: application/json 헤더를 설정하고, echo 후 exit합니다. 응답 형식: {"success": bool, "data": ..., "message": "..."}

전체 메서드 레퍼런스
메서드반환설명
ok(mixed $data = null, int $code = 200)never성공 응답 {"ok":true, "data":...}
fail(string $msg, mixed $details = null, int $code = 422)never실패 응답 {"ok":false, "message":...}
error(string $msg, int $code = 500)never에러 응답
paginated(array $data, int $total, int $page, int $perPage)never페이지네이션 응답 (meta 포함)
send(int $code, array $payload)never커스텀 JSON 응답
사용 예제
// 성공 응답
json()->ok(['user' => ['id' => 1, 'name' => 'Cat']]);
// → {"ok":true, "data":{"user":{"id":1,"name":"Cat"}}}

// 실패 응답 (details로 검증 오류 전달 가능)
json()->fail('이메일이 이미 존재합니다', null, 409);
// → {"ok":false, "message":"이메일이 이미 존재합니다"}

// 검증 오류 + 상세 정보 전달
json()->fail('유효성 검증 실패', $v->errors(), 422);

// 페이지네이션 응답
$posts = db()->table('posts')->limit(10)->offset(0)->all();
$total = db()->table('posts')->count();
json()->paginated($posts, $total, 1, 10);
주의: ok()/fail()/error()는 내부에서 exit을 호출합니다. 이후 코드는 실행되지 않으므로 핸들러의 마지막에 호출하세요.
관련: Api Paginate
