Queue

Cat\Queue — 비동기 작업 큐
queue()

Redis 또는 DB 기반 비동기 작업 큐입니다. 이메일 발송, 이미지 처리 등 시간이 걸리는 작업을 백그라운드에서 처리합니다.

push()로 작업을 큐에 추가하고, php cli.php queue:work로 워커를 실행합니다. later()로 지연 실행도 가능합니다.

전체 메서드 레퍼런스
메서드반환설명
handle(string $job, callable $handler)self작업 핸들러 등록
push(string $job, array $payload = [])string즉시 실행 큐에 추가 (ID 반환)
later(int $delay, string $job, array $payload)string지연 실행 (초 단위)
pop(?string $queue = null)bool단일 작업 꺼내기 + 실행
work(?string $queue, int $sleep, int $max)void워커 루프 (블로킹)
size(?string $queue = null)int큐 크기 조회
clear(?string $queue = null)int큐 비우기
failed(int $limit = 50)array실패한 작업 목록
retryFailed(string $id)bool실패한 작업 재시도
핸들러 등록 + 작업 추가
// 핸들러 등록 (cli.php 또는 부팅 시)
queue()->handle('send-email', function($payload) {
    mailer()->to($payload['to'])
        ->subject($payload['subject'])
        ->body($payload['body'])->send();
});

// 웹 핸들러에서 큐에 추가 (즉시 응답)
queue()->push('send-email', [
    'to' => 'user@example.com',
    'subject' => '환영합니다',
    'body' => '<h1>가입 완료</h1>',
]);

// 30초 뒤 실행
queue()->later(30, 'send-email', $payload);
워커 실행
# 터미널에서 워커 시작
php cli.php queue:work
php cli.php queue:size
php cli.php queue:failed
드라이버: redis(기본) 또는 database. Redis 드라이버는 blPop으로 효율적인 블로킹 대기를 합니다.
