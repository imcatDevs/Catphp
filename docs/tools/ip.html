Ip

Cat\Ip — 클라이언트 IP · GeoIP
ip()

클라이언트의 실제 IP 주소를 안전하게 감지하고, GeoIP 위치 정보(국가, 도시, 좌표)를 조회합니다. trusted_proxies 설정을 통해 신뢰할 수 있는 프록시/로드밸런서의 X-Forwarded-For 헤더만 참조하여 IP 스푸핑을 방지합니다.

프록시 환경에서는 X-Forwarded-For, X-Real-IP, CF-Connecting-IP(Cloudflare) 순으로 체크하며, 프록시가 없는 환경에서는 REMOTE_ADDR을 반환합니다. isInRange()로 IP가 CIDR 범위에 포함되는지 확인할 수 있습니다.

설정 — config/app.php
'ip' => [
    'trusted_proxies' => ['127.0.0.1', '10.0.0.0/8'],
]
전체 메서드 레퍼런스
메서드반환설명
address()string클라이언트 실제 IP 주소 반환
country(?string $ip = null)?string국가 코드 (예: 'KR')
city(?string $ip = null)?string도시명
location(?string $ip = null)?array위도/경도 ['lat'=>..., 'lon'=>...]
info(?string $ip = null)arrayGeoIP 전체 정보
isInRange(string $ip, string $cidr)boolIP가 CIDR 범위에 포함되는지 확인
기본 사용법
// ip()는 Cat\Ip 객체를 반환 — address()로 IP 문자열 획득
$clientIp = ip()->address();

// Rate limiting + 로깅에 활용
rate()->limit('api', 60, 100);
logger()->info('API 요청', ['ip' => ip()->address()]);
GeoIP 활용
// 국가별 분기 처리
$country = ip()->country();
if ($country === 'KR') {
    geo()->locale('ko');
}

// CIDR 범위 확인
if (ip()->isInRange(ip()->address(), '192.168.0.0/16')) {
    // 내부망 접속
}
보안: trusted_proxies를 설정하지 않으면 X-Forwarded-For 헤더를 무시하고 REMOTE_ADDR만 사용합니다. Cloudflare 뒤에 있다면 CF-Connecting-IP가 우선됩니다.
