Cat\Rate

파일 기반 레이트 리밋. 윈도우 시간 + 최대 횟수, flock 동시성 보호, Firewall 연동 자동 차단

// 60초 내 100회 제한
if (!rate()->limit('api', 60, 100)) {
    json()->error('Too Many Requests', 429);
}

// IP별 제한
rate()->limit('login:' . ip()->address(), 300, 5);

// 잔여 횟수 확인

public Cors
Cat\Cors

CORS 헤더 관리. 오리진/메서드/헤더 허용 설정, Preflight OPTIONS 자동 처리, Credentials 안전 제어

// 체이닝 설정 + 헤더 전송
cors()->origins(['https://example.com'])
     ->methods(['GET', 'POST', 'PUT', 'DELETE'])
     ->headers(['Content-Type', 'Authorization'])
     ->handle();

// 또는 config/app.php에서 설정 후 한 줄 호출
// 'cors' => ['origins' => [...], 'max_age' => 86400]
cors()->handle();
CORS 응답 헤더

api Api
Cat\Api

API 보안 미들웨어 통합. CORS + Rate Limit + Auth + Guard를 한 줄로 적용

// 한 줄로 4중 보안 적용
api()->cors()
     ->rateLimit(60, 100)  // 60초/100회
     ->guard()              // XSS/SQL 살균
     ->apply();             // 적용

// JWT 인증 추가
api()->cors()->rateLimit(60, 100)->auth()->guard()->apply();

// 실제 라우트 핸들러에서
router()->get('/api/users', function() {
    api()->cors()->rateLimit(60, 100)->guard()->apply();
    json()->ok(db()->table('users')->all());
});
미들웨어 파이프라인
1. CORS arrow_forward 2. Rate Limit arrow_forward 3. Auth (JWT) arrow_forward 4. Guard (XSS/SQL) arrow_forward 핸들러
각 미들웨어가 실패하면 즉시 에러 응답을 반환하고 다음 단계로 진행하지 않습니다.
